# Chinese Doc README

## 核心目标

- 用中文编写代码文档
- 用中文编写 README
- 严格保持代码语义和可执行命令不变

## 输出规则

- 所有自然语言注释与 docstring 使用简体中文
- README 标题与说明文字使用中文
- 除非用户明确要求，否则以下内容不得修改：
  - code blocks
  - shell commands
  - file paths
  - environment variable names
  - API field names and identifiers
- 翻译现有文件时只翻译说明性文字，保留原有结构与链接

## README 约定

新建或重写 README 时，优先使用以下章节顺序：

1. 项目简介
2. 功能特性
3. 环境要求
4. 安装步骤
5. 使用说明
6. 配置说明
7. 开发指南
8. 常见问题

各章节保持简洁、可执行。

## 代码文档约定

- 用中文说明意图、约束和副作用
- 避免“复述代码字面含义”的冗余注释
- 术语在同一项目中保持一致
- 保持现有格式、lint 风格和命名约定

## 执行流程

1. 识别目标文件，并区分 README、Markdown 文档、代码注释三类
2. 仅对自然语言文本应用中文化规则
3. 检查命令与代码示例仍可直接复制执行
4. 检查 Markdown 结构、链接和标题层级是否完整
5. 若任务包含功能改动，额外执行项目验证命令

## 触发示例

- “把这个项目的 README 改成中文”
- “代码注释和函数文档都用中文写”
- “把 docs 目录里的说明文档翻译为中文，但命令不要改”
